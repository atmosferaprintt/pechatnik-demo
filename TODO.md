# Отложенные задачи — CRM «ПЕЧАТНИК»

Всё, что «надо сделать потом» — складывается сюда.
Добавляется по команде пользователя **«Запланируй задачу на будущее»**.

## Формат

```
### YYYY-MM-DD · 🔴/🟡/🟢 · Краткое название
**Контекст:** почему задача появилась, какая проблема
**Что сделать:** конкретные шаги
**Блокеры / зависимости:** что мешает сделать сейчас
**Ориентир по времени:** 15 мин / 1 час / полдня
```

Приоритеты:
- 🔴 высокий — безопасность / потенциальная потеря данных / блокирует работу
- 🟡 средний — улучшения надёжности / UX / накопленный техдолг
- 🟢 низкий — nice-to-have / косметика

Самые свежие — снизу (хронологический порядок добавления).

## Задачи

### 2026-07-17 · 🟢 · Перетаскивание задач и заметок на тач-экранах (телефон/планшет)
**Контекст:** на доске задач и в «Заметках» (с 2026-07-19) перетаскивание сделано через HTML5 drag-and-drop — на тач-экранах эти события не срабатывают, работает только мышью/трекпадом.
**Что сделать:** тач-вариант через pointer events (долгое нажатие → тянуть карточку), с теми же dropInCol/dropOnGrid/sort_order.
**Блокеры / зависимости:** нет; сначала посмотреть, пользуются ли доской с телефона вообще.
**Ориентир по времени:** 1–2 часа

### 2026-07-17 · 🟡 · Баннер «вышло обновление» при новом деплое
**Контекст:** CRM — одностраничник; девочки держат вкладку открытой весь день и после деплоя сидят на старой версии, пока не обновят страницу руками (из-за этого «менеджера видят оплаты друг друга» уже после фикса).
**Что сделать:** при сборке класть в dist версию (хеш коммита) файлом `version.json`; клиент по фокусу вкладки/раз в N минут сравнивает со своей и показывает жёлтый баннер «Вышло обновление — нажми, чтобы обновить» (клик = location.reload()).
**Блокеры / зависимости:** нет.
**Ориентир по времени:** ~1 час

### 2026-07-21 · 🟡 · Депозиты и долги: менеджерам править записи при ошибке
**Контекст:** просьба Кристи: ошиблись в пополнении/списании депозита или записи должника — исправить нельзя; сейчас правится только имя, сами записи не редактируются ни у кого.
**Что сделать:** ✎/✕ на строках пополнений/списаний депозита и записях должника (что, сумма, дата): менеджеру — свои записи за сегодня/вчера, Кристи — любые; каждое исправление в журнал; RLS-миграция на deposit_uses / manual_debt_entries.
**Блокеры / зависимости:** «делай» Рабии; решить окно правки для менеджеров (как у операций — сегодня+вчера?).
**Ориентир по времени:** 2–3 часа с миграцией
